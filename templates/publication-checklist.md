# Public Skill Publication Checklist

Use this before publishing a local skill, automation, or workflow in a public repository.

## Privacy

- [ ] Remove credentials, API keys, tokens, cookies, and auth state.
- [ ] Remove customer, prospect, employee, partner, and private account data.
- [ ] Remove private URLs, workspace paths, schedule IDs, automation IDs, and internal labels.
- [ ] Replace real examples with fictional or anonymized examples.

## Portability

- [ ] Explain what tools or apps are required.
- [ ] Separate required setup from optional customization.
- [ ] Avoid assumptions that only work on one computer or one account.
- [ ] Link to related standalone repos when a workflow is too large for the hub.

## Quality

- [ ] State when to use the skill.
- [ ] State what inputs the skill needs.
- [ ] Include the expected output format.
- [ ] Include guardrails.
- [ ] Add at least one example prompt or usage scenario.

## Release Decision

- [ ] Keep as a summary in this hub.
- [ ] Publish the full skill inside this hub.
- [ ] Promote to a standalone repository.
